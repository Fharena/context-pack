# Context Pack v0.5.1

This security patch makes the repository a strict trust boundary for automatic context routing.

## Security Fixes

- Manifest paths containing traversal, absolute roots, Windows drive prefixes, UNC roots, NTFS stream syntax, or reserved device names are rejected before routing.
- Area documents are confined to the managed `AREAS/` directory.
- Automatic Evidence, search scopes, inference, and text measurement exclude ignored files and symbolic links.
- Managed Context Pack and agent-instruction files cannot redirect reads or writes through symbolic links.
- Non-Git discovery and legacy migration no longer recurse through linked directories.
- Packaged CLI validation now removes its npm, demo-repository, and plugin-install workspace on completion.

These checks prevent a repository-controlled manifest from reading an ignored local secret, reading a file outside the checkout, or redirecting a checkpoint/setup write outside its managed location.

## Compatibility

Normal relative manifest paths remain compatible. A manifest that intentionally points to ignored files, paths outside the repository, area documents outside `.context-pack/AREAS/`, or symbolic links now fails closed or omits that scope. Replace those entries with tracked, non-ignored repository files.

Explicit destinations supplied by the user, such as `--output`, remain user-controlled.

## Verification

- Cross-platform lexical tests cover traversal, POSIX roots, Windows drive-relative and absolute paths, UNC paths, and NUL bytes.
- CLI regressions verify that outside and ignored sentinel content is not emitted and that unsafe setup/checkpoint writes do not modify the target.
- Real symlink tests run where the host permits link creation; deterministic resolved-path tests run on every host.
- The final local suite passed 106 tests; two real-symlink cases were skipped on the Windows host because link creation was not permitted and remain active on Ubuntu CI.
- The installed npm tarball rejected an outside-path sentinel fixture, and the wheel/sdist passed `twine check`; the wheel installed and reported `context-pack 0.5.1` in an isolated D-drive venv.
- The deterministic public routing gate passed 10/10 scenarios with a matching handoff replay signature after segment-aware scope matching was restored.
