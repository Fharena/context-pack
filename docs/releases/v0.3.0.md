# Context Pack v0.3.0

Context Pack now behaves like quiet agent infrastructure instead of a setup wizard hidden inside normal work.

## Highlights

- Safe transient first run: `start` no longer creates `.context-pack/`, `AGENTS.md`, or `.gitignore` without explicit setup.
- Smaller-project escape hatch: unconfigured repositories with 24 files or fewer skip pack generation.
- Portable Git handling: UTF-8/NUL-delimited parsing, exact-interpreter hooks, preserved Unix absolute paths, and safe fail-open behavior.
- Generic routing: custom area roles and bounded entry-point globs replace benchmark-specific buckets and whole-directory starts.
- Lower steady-state cost: normal `start` avoids repository-wide text-budget scans, logs rotate, and `doctor` warns about overly broad areas.
- Single-source packaging: the engine, skill, agent metadata, and plugin manifest are synchronized and checked in CI.
- Leaner surface: natural-language intent stays with the agent; default CLI help and public docs focus on setup, start, review, checkpoint, and doctor.

## Evidence

The reproducible orientation benchmark passed all 10 public-repository scenarios. Approximate first-read ratios ranged from 4% to 65%; BrowserQuest scenarios routed at 16%, and a fresh-clone handoff replay reproduced the same 12% routing signature.

These are `chars/4` text-budget estimates, not billed tokens, and they do not prove patch-quality improvements. See [`docs/BENCHMARKS.md`](../BENCHMARKS.md).

## Upgrade

Existing repositories remain compatible. Run:

```bash
context-pack doctor --fix
```

Existing unbounded logs are compacted on the next checkpoint. Generated `REVIEW.md` and `CONTRACTS.md` files are no longer used and may be removed.
