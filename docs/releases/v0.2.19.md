# Context Pack v0.2.19

This release improves first-run routing for medium web game and client/server JavaScript repositories, then documents a reproducible A/B orientation benchmark.

## Highlights

- Recognizes common web source layouts such as `client/js`, `server/js`, and `shared/js`.
- Splits game/frontend assets into `sprites`, `maps`, and `media` instead of routing every asset prompt through one broad bucket.
- Avoids vendored `client/js/lib` files in initial read-first globs.
- Adds BrowserQuest A/B orientation results:
  - broad repo text: ~602.2k tokens
  - Context Pack first-read: ~98.2k-103.6k tokens
  - task-specific first-read budget: ~16-17% of repo text

## Verification

- `python -m unittest discover -s tests -v`
- `python scripts/validate_packaged_cli.py`
- `node bin/context-pack.js --help`
- `npm pack --dry-run`
- `python -m build`
- `python -m twine check dist/*`

GitHub Release assets include the Python wheel, Python sdist, and npm tarball. Registry publishing remains opt-in and requires PyPI/npm trusted publishing or local auth.
