# Context Pack v0.4.0

Context Pack now routes agents to patch-ready source evidence instead of stopping at search suggestions.

## Highlights

- `start --agent` emits one compact response with at most two line-numbered source regions.
- Configured symbols run before broad task words; fallback task search is used only when symbols are unavailable.
- Agent packs cap routing at two areas, four scopes, three contracts, two failure modes, and one verification command.
- Contract and failure prose no longer selects unrelated areas.
- Equivalent canonical and bundled source snippets are deduplicated.
- Agent rules avoid reopening shown source ranges and reserve checkpoints for meaningful session boundaries.
- The Codex A/B harness records model turns and tool-output size and forces trials to use the engine under test.

## Evidence

On the five-run BrowserQuest zoning task, baseline and curated conditions both produced the correct minimal patch in 5/5 runs.

| Median | Baseline | Evidence-first curated | Change |
| --- | ---: | ---: | ---: |
| Total input | 111,828 | 68,075 | 39.1% less |
| Uncached input | 20,948 | 6,905 | 67.0% less |
| Commands | 4 | 5 | 25.0% more |
| Tool output chars | 47,809 | 4,298 | 91.0% less |
| Duration | 38.2s | 43.7s | 14.3% slower |

This is one seeded task with curated project knowledge, not a universal cost, latency, or productivity claim. Lower token volume did not produce lower latency in this batch. See [`docs/BENCHMARKS.md`](../BENCHMARKS.md) and the [machine-readable result](../benchmarks/codex-ab-zoning-evidence.json).

## Upgrade

Existing repositories remain compatible. Refresh managed agent rules with:

```bash
context-pack doctor --fix
```

No manifest migration is required. Add precise `search_terms` and one high-value verification command to important areas to benefit from evidence-first routing.
