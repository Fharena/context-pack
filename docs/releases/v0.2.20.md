# Context Pack v0.2.20

This release expands Context Pack's benchmark coverage and hardens weak spots found while testing public repositories.

## Highlights

- Adds a reproducible benchmark harness for public-repo orientation, threshold checks, slow-measure flags, and handoff replay.
- Refreshes benchmark docs with 10 public scenarios across Python, JavaScript, web game assets, Go, Rust, CI/build, review-like debugging, and test-failure prompts.
- Improves Go first-run inference for root `*.go`, top-level package directories, and `*_test.go` files.
- Improves Rust/filter/search routing through common crate paths and source keywords.
- Skips known binary media/archive/font files before text-budget reads and checks file size before reading, reducing cold-start work on asset-heavy repos.

## Benchmark Snapshot

- Public scenarios: 10
- Weak public scenarios: 0 under the current benchmark thresholds
- BrowserQuest web game tasks: ~16-17% first-read text budget versus broad repo text
- `gin-gonic/gin` middleware prompt: `source`, ~4% first-read text budget
- `expressjs/express` router prompt: `source`, ~9% first-read text budget
- `sharkdp/fd` regex filter prompt: `source`, ~30% first-read text budget
- Handoff replay: same routing signature after a local clone

These are deterministic orientation and text-budget checks, not independent-agent patch-quality claims.
