# Context Pack v0.5.0

Context Pack now distinguishes maintained project knowledge from cold-start guesses and reports how much confidence an agent should place in each source region.

## Highlights

- Review routing loads manifests and area notes from the base commit, so a branch cannot define the contracts used to review itself.
- Evidence is labeled `strong`, `candidate`, or absent according to provenance.
- `start --agent` without a task resumes from `CURRENT.md` instead of inventing a generic continuation task.
- Areas have explicit `verify` commands; `doctor` checks missing, duplicated, and overly broad search symbols.
- `setup --agent-docs auto` manages `AGENTS.md` and refreshes Claude/Cursor files only when those environments are detected.
- The installed skill automatically routes configured repositories only. Unconfigured repositories use normal targeted search unless a transient preview is explicitly requested.

## Evidence

The v0.5.0 Codex CLI harness covers four task classes on a pinned BrowserQuest revision.

| Task | Baseline correct | Maintained correct | Total-input change | Uncached-input change |
| --- | ---: | ---: | ---: | ---: |
| Precise bug | 3/3 | 3/3 | 29.4% less | 41.6% less |
| Domain-routed bug | 5/5 | 5/5 | 37.5% less | 44.1% less |
| Branch review | 3/3 | 3/3 | 16.5% less | 44.1% less |
| Session continuation | 3/3 | 3/3 | 21.3% less | 48.7% less |

Maintained context and baseline were both correct in 14/14 runs, while experimental transient routing was correct in 11/14. Transient routing also increased median total input by 14.1% on the domain task and 41.3% on continuation. Those negative results drove the safer default.

This remains a small author-run benchmark on one repository with seeded defects. It is not a universal token, billing, latency, or quality claim. Read the [method and limits](../BENCHMARKS.md) before quoting the figures.

## Upgrade

Existing manifests remain compatible. To refresh managed agent rules:

```bash
context-pack doctor --fix
```

For important areas, move executable commands from `tests` into `verify` and keep `tests` for test files or scopes. Existing `tests` command entries still work as a compatibility fallback.

Use `context-pack setup --agent-docs all` if you intentionally want to create `AGENTS.md`, `CLAUDE.md`, and Cursor rules together. The default `auto` mode avoids unused agent files.
